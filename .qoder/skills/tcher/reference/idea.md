"Idea" is the freeform door into tcher: one short brief in, a few genuinely different directions back. The failure mode is three variations of the same safe layout (same palette, same hero-left-copy-right skeleton, a different border radius). Reject that first. A direction is a different bet on hierarchy, type voice, color world, or structure, not a different coat of paint.

---

## Register

Brand: directions can sit far apart: editorial, technical, warm-consumer. PRODUCT.md's stated voice is the ceiling on how far to roam, not the floor.

Product: directions vary within earned familiarity. Density, emphasis, structure, and rhythm move; the interaction grammar users already know stays put.

---

## Process

1. **Read the brief literally, then once more for intent.** "A pricing page" carries an implied audience and a conversion job; name both before sketching anything.

2. **Anchor on context.** PRODUCT.md and DESIGN.md (or tokens extracted from the code when DESIGN.md is missing) decide which identity to preserve. Departing from the existing identity needs an explicit trigger in the brief, not your own boredom with it.

3. **Commit to 2-3 distinct directions.** Write each direction's identity sentence first: what it bets on, what it sacrifices. If two sentences read alike, rework before building. Distinctness lives on the primary axes (structure, type, color commitment, density), never on decoration.

4. **Build the strongest direction fully.** Sketch the others only as far as the user needs in order to choose. When the brief does not pick, ask which direction to pursue rather than shipping three half-built ones.

5. **Present with trade-offs stated out loud.** Density vs scannability, personality vs neutrality, motion vs calm. A direction without a named cost has not been thought through.

When the user gives an existing target (`/tcher idea checkout`), treat the current implementation as direction zero: diagnose what it bets on now, then propose directions that bet differently.

In live mode this command's spirit drives the **Idea** action (`action: "tcher"`): no sub-command reference is loaded there; variants come from the shared design laws plus the register reference, per reference/live.md.
